Compiling and Running BPF Programs

1. Prepare the Environment

Ensure you have the following tools and libraries installed:
LLVM and Clang (version 10 or later recommended)
libelf-dev
Linux headers for your kernel version
bpftool (usually comes with the Linux kernel source)
On Ubuntu or Debian-based systems, you can install these with:
Shell
Copy
sudo apt-get update sudo apt-get install llvm clang libelf-dev linux-headers-$(uname -r) bpftool
​

2. Compile the BPF Program

Save your BPF program to a file, e.g., ip_id_reflect.c
Compile the program using Clang:
Shell
Copy
clang -O2 -target bpf -c handle_tcp_hdr_opt.c -o handle_tcp_hdr_opt.o
​

3. Load and Pin the BPF Program

Create the BPF filesystem if it's not already mounted:
Shell
Copy
sudo mount -t bpf bpf /sys/fs/bpf/
​
Load and pin the BPF program using bpftool:
Shell
Copy
sudo bpftool prog load ip_id_reflect.o /sys/fs/bpf/ip_id_reflect
​
This command loads the program and pins it to /sys/fs/bpf/ip_id_reflect.

4. Attach the BPF Program

To attach the program to all cgroups (system-wide):
Shell
Copy
sudo bpftool cgroup attach /sys/fs/cgroup/unified/ip_id_reflect sock_ops pinned /sys/fs/bpf/ip_id_reflect
​
Or, to attach to a specific cgroup (e.g., for a specific application):
Create a cgroup (if not already existing):
Shell
Copy
sudo mkdir -p /sys/fs/cgroup/ip_id_reflect
​
Attach the program to this cgroup:
Shell
Copy
sudo bpftool cgroup attach /sys/fs/cgroup/unified/ip_id_reflect sock_ops pinned /sys/fs/bpf/ip_id_reflect
​

5. Testing the Program

If you're using a specific cgroup, you need to run your target application in that cgroup:
Find the PID of your application:
Shell
Copy
pidof your_application
​
Add the application to the cgroup:
Shell
Copy
echo PID_OF_YOUR_APPLICATION > /sys/fs/cgroup/ip_id_reflect/cgroup.procs
​

6. Monitoring and Debugging

Use sudo bpftool prog trace to see if your program is being hit.
Check the kernel logs (dmesg) for any debug output from your program.
Use tools like tcpdump or Wireshark to verify if the IP ID is being set correctly in outgoing packets.

7. Cleanup

To remove the BPF program:
Detach the program from the cgroup:
Shell
Copy
sudo bpftool cgroup detach /sys/fs/cgroup/unified/ip_id_reflect sock_ops pinned /sys/fs/bpf/ip_id_reflect
​
(If you attached to a specific cgroup, replace / with the cgroup path)
Remove the pinned program:
Shell
Copy
sudo rm /sys/fs/bpf/ip_vni_reflect
​
If you created a specific cgroup, remove it:
Shell
Copy
sudo rmdir /sys/fs/cgroup/ip_id_reflect
​
Note: These instructions assume you're running on a relatively recent Linux system with BPF support. The exact commands might vary depending on your specific system configuration.
Note: Using tcpdump requires root privileges. Be cautious when running these commands, especially on production systems.
