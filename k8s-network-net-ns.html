Net NS

Introduce

实践 step by step

创建一个net ns
Shell
Copy
ip netns add netns-duheng
​
此时可以进入netns中查看，里面只有一个会环网卡,并且是down，且ping127.0.0.1是不通的
Shell
Copy
$ sudo ip netns exec netns-duheng ip link 1: lo: <LOOPBACK> mtu 65536 qdisc noop state DOWN mode DEFAULT group default qlen 1000 link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
​
我们需要将这个网卡启用。然后就可以ping通
Shell
Copy
# 在netns中将lo这个网卡启用 sudo ip netns exec netns-duheng ip link set dev lo up # 然后再ping 就可以通了 sudo ip netns exec netns-duheng ping 127.0.0.1 PING 127.0.0.1 (127.0.0.1) 56(84) bytes of data. 64 bytes from 127.0.0.1: icmp_seq=1 ttl=64 time=0.041 ms 64 bytes from 127.0.0.1: icmp_seq=2 ttl=64 time=0.062 ms
​
此时，netns中的网络只是回环能够通，如果想要和主机互通（跨namespace）,需要使用到veth pair，（具体介绍请到TODO）
Shell
Copy
# 在主机上 sudo ip link add veth0 type veth peer name veth1 # 可以看到出现了两veth 设备 $ ip link 1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN mode DEFAULT group default qlen 1000 link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00 29: veth1@veth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc noqueue state UP mode DEFAULT group default qlen 1000 link/ether 72:3f:ee:db:1b:7b brd ff:ff:ff:ff:ff:ff 30: veth0@veth1: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc noqueue state UP mode DEFAULT group default qlen 1000 link/ether 66:af:a4:08:4b:b7 brd ff:ff:ff:ff:ff:ff
​
然后我们将另veth1 连接到netns中，此时看到网卡少了一个（被连接到netns中了）
Shell
Copy
$ sudo ip link set veth1 netns netns-duheng # 主机上 $ ip link 1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN mode DEFAULT group default qlen 1000 link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00 30: veth0@if29: <NO-CARRIER,BROADCAST,MULTICAST,UP> mtu 1500 qdisc noqueue state LOWERLAYERDOWN mode DEFAULT group default qlen 1000 link/ether 66:af:a4:08:4b:b7 brd ff:ff:ff:ff:ff:ff link-netns netns-duheng $ sudo ip netns exec netns-duheng ip link 1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN mode DEFAULT group default qlen 1000 link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00 29: veth1@if30: <BROADCAST,MULTICAST> mtu 1500 qdisc noop state DOWN mode DEFAULT group default qlen 1000 link/ether 72:3f:ee:db:1b:7b brd ff:ff:ff:ff:ff:ff link-netnsid 0
​
此时两veth peer 的两端还是down的，我们需要为其配置ip，并且将其up，此时就能ping 通配置的ip了
Shell
Copy
sudo ip netns exec netns-duheng ifconfig veth1 192.168.1.1/24 up $ sudo ip netns exec netns-duheng ip link 1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN mode DEFAULT group default qlen 1000 link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00 29: veth1@if30: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc noqueue state UP mode DEFAULT group default qlen 1000 link/ether 72:3f:ee:db:1b:7b brd ff:ff:ff:ff $ sudo ip netns exec netns-duheng ping 192.168.1.1 PING 192.168.1.1 (192.168.1.1) 56(84) bytes of data. 64 bytes from 192.168.1.1: icmp_seq=1 ttl=64 time=0.030 ms 64 bytes from 192.168.1.1: icmp_seq=2 ttl=64 time=0.045 ms 64 bytes from 192.168.1.1: icmp_seq=3 ttl=64 time=0.042 ms
​
同时将主机上的veth0也配置ip并且启用,也能ping通了，这样就实现了跨netns的通信了
Shell
Copy
sudo ifconfig veth0 192.168.1.2/24 up $ ping 192.168.1.2 PING 192.168.1.2 (192.168.1.2) 56(84) bytes of data. 64 bytes from 192.168.1.2: icmp_seq=1 ttl=64 time=0.028 ms 64 bytes from 192.168.1.2: icmp_seq=2 ttl=64 time=0.048 ms 64 bytes from 192.168.1.2: icmp_seq=3 ttl=64 time=0.049 ms 64 bytes from 192.168.1.2: icmp_seq=4 ttl=64 time=0.046 ms 64 bytes from 192.168.1.2: icmp_seq=5 ttl=64 time=0.048 ms
​
